## 1. 프로젝트 개요

### 1.1 목적
차수별로 변동되는 참석자의 문서를 효율적으로 수집하고 관리하며, 마감 기한에 맞춰 리마인드 알림을 제공하여 제출률을 높이는 것을 목표로 한다.

### 1.2 주요 기능
*   차수 관리 (생성/수정/삭제)
*   차수별 참석자 선택
*   문서 제출 (파일 업로드)
*   제출 현황 대시보드
*   ZIP 파일 다운로드
*   SMS 및 이메일 알림 (마감일 3일 전, 1일 전, 당일)

### 1.3 사용자
*   관리자: 차수 관리, 사용자 관리, 제출 현황 확인
*   참여자: 문서 제출, 제출 현황 확인

## 2. 상세 기능

### 2.1 차수 관리
*   관리자는 차수를 생성, 수정, 삭제할 수 있습니다.
*   차수 생성 시 참석자를 선택하고, 필수 제출 문서를 설정할 수 있습니다.

### 2.2 문서 제출
*   참여자는 본인이 참석하는 차수를 확인하고, 해당 차수에 필요한 문서를 제출할 수 있습니다.
*   파일 업로드 또는 "해당 없음" 사유를 제출할 수 있습니다.

### 2.3 알림 기능
*   마감일 3일 전, 1일 전, 당일에 SMS 및 이메일 알림을 발송합니다.
*   알림에는 차수 정보, 제출해야 할 문서 목록, 제출 링크가 포함됩니다.

### 2.4 제출 현황 대시보드
*   관리자는 차수별 제출 현황을 한눈에 파악할 수 있습니다.
*   참여자의 제출 상태 (완료, 미제출)를 확인하고, 미제출자에게 알림을 보낼 수 있습니다.

### 2.5 ZIP 파일 다운로드
*   관리자는 차수별 제출 문서를 ZIP 파일로 다운로드할 수 있습니다.

## 3. 사용자 여정

### 3.1 관리자
1.  로그인
2.  차수 목록 확인
3.  새로운 차수 생성 또는 기존 차수 수정
4.  참석자 선택 및 필수 문서 설정
5.  제출 현황 대시보드에서 제출 상황 확인
6.  필요시 미제출자에게 SMS 또는 이메일 알림 발송
7.  문서 취합 완료 후 ZIP 파일 다운로드

### 3.2 참여자
1.  로그인
2.  본인이 참석해야 하는 차수 목록 확인
3.  해당 차수에 필요한 문서 제출 (파일 업로드 또는 "해당 없음" 사유 제출)
4.  제출 완료 후 제출 현황 확인

## 4. 추가 고려 사항
*   사용자 인터페이스는 직관적이고 사용하기 쉬워야 합니다.
*   모바일 환경에서도 원활하게 작동해야 합니다.
*   알림 메시지는 명확하고 간결해야 합니다.
